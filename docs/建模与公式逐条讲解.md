## 4.1 系统模型与符号

### 4.1.1 时间离散化

> 将连续时间离散为时隙 $t \in \{0, 1, 2, \dots\}$,时隙长度 $\Delta t$(典型 1 s)。

**为什么离散化:**

- 卫星网关决策是**事件触发型**的(看到新可见性变化 → 重新选),不是连续控制问题,离散化损失可忽略。
- 离散化让 Lyapunov 漂移-罚函数推导直接落到求和而不是积分,数学更清爽。

**为什么 $\Delta t = 1\,\text{s}$:**

- LEO 卫星地面速度 ~7.6 km/s,1 s 移动 7.6 km,远小于 ISL 通信范围 $D_{\max} = 3500$ km(约 0.2% 移动),所以链路状态在 1 s 内可近似为不变。
- 1 s 也远大于 `decision_fn` 的执行时间(< 1 ms),保证决策时间富裕、且为后续两阶段迁移留出预算。

**与代码对应:** `config.py:SimConfig.decision_interval_sec = 1.0`。

---

### 4.1.2 几何与链路量(符号表)

| 符号 | 定义 | 怎么算出来 | 建模依据 |
|---|---|---|---|
| $D_i(t)$ | $s$ 与 $g_i$ 的欧氏距离(km) | $\|\mathbf{p}_s(t) - \mathbf{p}_{g_i}(t)\|_2$,位置由 SGP4 从 TLE 外推 | 物理距离决定传播延迟和带宽,是 ISL 最基本量 |
| $\theta_i(t)$ | ISL 链路与最近地平切线的**余隙角**(grazing angle,°) | 见 `orbit/skyfield_sim.py:elevation_deg()` | **不是地面站仰角**,而是两端卫星视线相对地心切线的最小余隙;$\theta < 0$ 表示连线被地球挡住 |
| $\theta_{\min}$ | 最小通信仰角(默认 10°) | 工程经验值 | 综合考虑大气衰减、地球扁率、天线方向图等的总裕度 |
| $D_{\max}$ | ISL 最大距离(默认 3500 km) | 激光 ISL 链路预算上界 | 距离更远会信噪比不足,无法解调 |
| $V_i(t)$ | 二值可见性指示 ∈ {0, 1} | 见 [4.1.3](#413-可见性公式) | 把"能不能通信"这个布尔事实抽象出来,后面所有决策都基于它 |
| $\Delta T_i(t)$ | 剩余连续可见时长(s) | 沿星历向前扫到第一个 $V=0$ 的时刻 | **本论文的关键变量**——把"未来可知"这个卫星场景特性显式建模出来,是地面网络无法享受的优势 |
| $B_i(t)$ | ISL 带宽(Mbps) | $B(D) = B_{\max} - \frac{D-D_{\min}}{D_{\max}-D_{\min}}(B_{\max}-B_{\min})$,见 `isl_bandwidth_mbps()` | 距离越大,激光发散损耗越大,信道容量降低;线性模型是工程上的常用简化 |
| $L_i(t)$ | 网关 CPU 协议转换负载 ∈ [0, 1] | 仿真生成:`base + 慢正弦 + 噪声`,见 `synthesize_gateway_loads()` | 占用率影响协议转换吞吐(`gateway_cpu_capacity_pps × (1-L)`),决策时要避开高负载者 |
| $\tau_i(t)$ | 传播延迟(ms) | $\tau = D / c \times 10^6$(光速)| 纯几何量,直接影响 E2E 延迟 |
| $a(t)$ | 决策变量 ∈ {1, ..., N} | 算法输出 | 当前时隙服务该 AOS 流的网关 ID |

> **提醒**:ISL 的"可见性"由两端**地心矢量与连线的几何切线**判定(line-of-sight, LOS),与地面站仰角的语义完全不同。代码实现见 `orbit/skyfield_sim.py:has_line_of_sight()`。

### 4.1.3 可见性公式

$$V_i(t) = \begin{cases} 1, & \theta_i(t) \ge \theta_{\min} \;\wedge\; D_i(t) \le D_{\max} \\ 0, & \text{otherwise} \end{cases}$$

**含义:** 网关 $g_i$ 在 $t$ 时刻对 AOS 可见当且仅当**两个条件都满足**:

1. **几何角度**:视线与地球切线夹角 $\ge 10°$ — 不被地球或大气阻挡
2. **物理距离**:不超过 $D_{\max} = 3500$ km — 信噪比足以建链

**为什么是合取 (AND) 而不是析取 (OR):** 任何一个不满足都不能建链——夹角太小被地球挡住,大距离则激光发散到不可解调。两个条件**物理独立但同时必需**。

**为什么用二值而不是连续质量分数:**

- 二值可直接进入指示函数 $\mathbf{1}\{\cdot\}$,推导和实现更简洁
- 部分可见的"软"状态在 ISL 上极短(< 1 s 即由可见跳到不可见),离散化代价微乎其微
- 若后续需要 QoS 分级,可在 $\theta$ 的基础上加分段函数,但本工作没必要

**与代码对应:** `orbit/topology.py:TopologySnapshot.visible[t_idx, j]`(预计算好的 $T \times N$ 张量)。

### 4.1.4 剩余可见时长

$$\Delta T_i(t) = \inf\{ \Delta : V_i(t+\Delta) = 0 \} - t,\quad \text{若 } V_i(t) = 1$$

**含义:** 从当前时刻 $t$ 起,$g_i$ 还能持续可见多少秒。若现在不可见 ($V_i(t) = 0$),约定 $\Delta T_i(t) = 0$。

**符号拆解:**

- $\inf\{\Delta : V_i(t+\Delta) = 0\}$ — 从 $t$ 起,**第一次**变为不可见的时刻(下确界 = 最早时间)
- $-t$ — 减去当前时刻,得到剩余时长

**为什么这个变量是核心:**

- 它告诉算法"**这条链路还能用多久**",可以提前 $T_h$ 秒就标记当前网关有问题、触发预迁移
- **地面 RAT(MPTCP、DRL)只能用过去信号强度做事后判断,本论文用 $\Delta T_i$ 显式利用了"未来可知"这一卫星场景独有特性**
- 这是 Lyapunov 在线决策与 DRL 在卫星场景下不同的本质差异

**与代码对应:** `orbit/topology.py:remaining_visibility(t_idx)`——直接沿离散时隙向前扫描 `visible[t_idx:, j]` 找第一个 False 的位置:

```python
def remaining_visibility(self, t_idx: int) -> np.ndarray:
    rem = np.zeros(self.num_gateways, dtype=float)
    for j in range(self.num_gateways):
        if not self.visible[t_idx, j]:
            continue
        k = t_idx
        while k < self.num_steps and self.visible[k, j]:
            k += 1
        rem[j] = (k - t_idx) * dt
    return rem
```

### 4.1.5 决策变量与即时代价

$$a(t) \in \{1, 2, \dots, N\}$$

$$c(t) = \alpha \cdot \underbrace{\mathbf{1}\{V_{a(t)}(t) = 0 \;\vee\; \Delta T_{a(t)}(t) < T_h\}}_{\text{中断指示}} + \beta \cdot L_{a(t)}(t) + \gamma \cdot \mathbf{1}\{a(t) \ne a(t-1)\}$$

**这是整个 Lyapunov 框架的优化对象**,逐项展开:

| 项 | 数学表达 | 物理含义 | 默认权重 |
|---|---|---|---|
| 第 1 项 中断惩罚 | $\alpha \cdot \mathbf{1}\{V_a = 0 \vee \Delta T_a < T_h\}$ | 若所选网关不可见,或剩余可见时长 < $T_h = 5$ s,加 $\alpha$ 的惩罚 | $\alpha = 1.0$ |
| 第 2 项 负载惩罚 | $\beta \cdot L_a$ | 与选中网关的当前负载线性正相关 | $\beta = 0.3$ |
| 第 3 项 切换瞬时代价 | $\gamma \cdot \mathbf{1}\{a(t) \ne a(t-1)\}$ | 若本时隙改了网关,加 $\gamma$ | $\gamma = 0.5$ |

**权重选取的依据:**

- $\alpha = 1.0$ 是 **baseline 标尺**——中断是最严重事件,其它权重以 $\alpha$ 为参照
- $\beta = 0.3 < \alpha$——负载是渐变量,不应主导决策;$\beta \cdot L \in [0, 0.3]$ 总小于 $\alpha = 1$,保证中断惩罚永远优先
- $\gamma = 0.5$——切换是离散一次性事件,代价介于"满载工作"和"中断"之间;且 $\gamma > \beta \cdot \max L = 0.3$ 保证**不必要的负载小波动不会触发切换**

**为什么用 $T_h = 5$ s 的阈值,而不是只看 $V = 0$:**

- 如果只看当前 $V$,等到不可见才换 = 被动硬切换 → 中断已发生
- 加 5 s 预警让算法**提前** $T_h$ 秒就标记当前网关有问题,触发两阶段迁移
- 5 s > `pre_copy_lead_sec = 3` s,保证 Pre-copy 在断链前能完成

**与代码对应** (`optimizer/lyapunov_solver.py`):

```python
interrupt = (not vis_k[a]) or (rem[a] < interruption_horizon_sec)
cost_a = alpha * (1.0 if interrupt else 0.0) + beta * L_k[a]
realized_cost = cost_a + gamma_switch * (1.0 if is_switch else 0.0)
```

## 4.2 翻译上下文(TC)数据结构

### 4.2.1 静态/动态二分的依据

$$C = C_{static} \cup C_{dynamic}$$

**核心依据:迁移代价 vs 变化频率二象限**

| | 变化慢 | 变化快 |
|---|---|---|
| **代价小** | 几乎不迁移就行 | 频繁增量同步 |
| **代价大** | **$C_{static}$ — Pre-copy 提前推** | **$C_{dynamic}$ — Stop-and-copy 关键时刻冻结** |

把字段按这个分类后,自然得到 **Pre-copy + Stop-and-copy 的两阶段调度**——这是创新点三的核心结构性发现,不是从 VM 迁移照搬来的,而是从协议级状态的固有性质推出的。

---

### 4.2.2 $C_{static}$:静态映射表

```
StaticContext
  mappings: {(scid, vcid) → IPv6Mapping}   # 协议翻译规则
  qos_dscp, bandwidth_quota_kbps             # QoS 策略
  gateway_ipv6_addr                          # 网关本身的源 IPv6
  aos_scid_list                              # 该网关服务的 AOS 卫星列表
```

**字段含义:**

- `(scid, vcid)`:CCSDS 标识符——`scid` 是航天器 ID,`vcid` 是虚拟信道 ID,共同标定一条业务流
- `IPv6Mapping`:目标 IPv6 地址、端口、DSCP(QoS 码点)、带宽配额

**为什么是"静态":**

- 这些规则在网关运行期间几乎不变(对应论文创新点二),改动只发生在 AOS 卫星上线/下线时(分钟到小时尺度)
- 在切换的几秒尺度上,可以视为常量
- **这意味着可以提前 $t_{pre}$ 秒推送过去**,不影响在线服务

**实测大小:** 单 AOS 会话约 288 B(`results/summary.json` 中 `static_bytes_replicated = 288`),远小于 ISL 带宽 × pre_copy_lead = $100\,\text{Mbps} \times 3\,\text{s} \approx 37.5\,\text{MB}$ 的预算 → Pre-copy 100% 能完成。

**关键方法:**

- `lookup(scid, vcid)`:查询翻译规则(协议转换主路径)
- `size_bytes()`:估算传输字节数(给 Pre-copy 算时间用)
- `fingerprint()`:SHA1 哈希前 12 字符,用于副本一致性比对

---

### 4.2.3 $C_{dynamic}$:动态运行时状态

```
DynamicContext
  frag_buffer: FragmentReassemblyBuffer
    buf:           {(scid, vcid, ccsds_pkt_id) → [CCSDSFragment]}
    _complete_set: {(scid, vcid, ccsds_pkt_id)}  ← v2 增量集合
    last_update_sec
  queues: {(scid, vcid) → VCIDQueueState}
  version, timestamp_sec
```

#### `buf`:分片重组缓存(创新点三相对 VM 的最大独特性)

**键 / 值定义:**

- 键 `(scid, vcid, ccsds_pkt_id)`:某条业务流 (scid, vcid) 中第 `ccsds_pkt_id` 号 CCSDS 包
- 值 `[CCSDSFragment]`:已收到的若干分片列表

**为什么需要这个缓存:**

- 1 个 CCSDS 包通常分 3-8 片放入若干 AOS 帧的 M_PDU
- 网关必须等所有分片到齐才能重组成完整 CCSDS 包再封 IPv6 → 切换瞬间未完成的分片就是协议级状态
- **VM Live Migration 没有这个概念**——VM 应用层透明,不知道下层有"协议分片"

**判定一个 CCSDS 包是否完成:**

$$\text{Complete}(k) \;\iff\; |\text{buf}[k]| = \text{buf}[k][0].\text{frag\_total}$$

(即收到的分片数 = 该包的总分片数)

#### `_complete_set`:v2 增量优化字段

- 维护**已收齐的 key 集合**,避免每帧扫描全 `buf`
- 在 `add()` 中,每次往 `buf[k]` 追加分片后判断是否刚好达到 `frag_total`,若达到则 `_complete_set.add(k)`
- `complete_packets()` 直接返回这个集合,**时间复杂度从 O(buf 大小) 降到 O(完成集合大小)**
- 单次仿真因此 **200× 提速**(451 s → 2.3 s),详见 `results/EXPERIMENT_REPORT.md` 第 8 节

#### `queues`:VCID 级 QoS 队列状态

```
VCIDQueueState
  queue_len, queued_bytes      # 当前队列长度/累计字节
  last_dequeue_sec             # 上次出队时间(QoS 调度用)
  sequence_no                  # 已发送出去的 IPv6 报文序号水位
```

**为什么需要迁移:**

- 切换后新网关如果不知道序号水位,会重发已发送的报文或乱序
- 高优先级 VCID(vcid ≤ 1)的队列必须保序,否则上层应用感受到丢包/乱序

#### `version` 字段(Lamport 时钟思想)

- 单调递增计数器,在 `bump_version()` 时 +1
- 用于 Gossip 一致性协议判断"谁是更新的副本"
- 多网关并发覆盖时,新版本的副本可以让旧版本失效,避免脑裂

### 4.2.4 关键方法及其设计理由

| 方法 | 用途 | 设计理由 |
|---|---|---|
| `snapshot()` 深拷贝 | Pre-copy 拍快照 | 后续要算 $\Delta C_{dynamic}$ 增量,需要冻结的参考点 |
| `diff_bytes_since(prev)` | 估算增量字节 | Stop-and-copy 只传 $\Delta$ 而非全量,大幅缩短同步时间 |
| `complete_packets()` | 找已重组完的包 | 每个时隙都要扫,是性能热点(v2 已优化) |
| `num_partial()` | 数未完成分片 | 硬切换路径下,此数 = 净丢失分片数 |
| `fingerprint()` | 静态上下文摘要 | Gossip 比对用,12 字节 SHA1 前缀足够区分 |

## 4.3 Lyapunov drift-plus-penalty 决策

### 4.3.1 优化目标

$$\min_{\{a(t)\}} \;\; \overline{c} \;\;=\;\; \lim_{T\to\infty} \frac{1}{T}\sum_{t=0}^{T-1} c(t)$$

**含义:** 在所有可能的决策序列 $\{a(0), a(1), \dots\}$ 中找一条,让**长期时间平均代价** $\overline{c}$ 最小。

**为什么是长期平均而不是瞬时:**

- **瞬时最小** → 每步贪心选当下最便宜的 → 会反复切换造成累计开销(反例见 MPTCP-style 实测 70 次切换/30 min)
- 卫星运行是 7×24 长时尺度,平均量更符合用户感受
- 数学上长期平均允许"短期亏损换长期收益"——例如现在保持高负载网关,换取下次切换不必那么频繁

**为什么不直接最小化某次切换的代价:**

- 单次代价最小 = 永远不切换 = 当前网关不可见后中断(违背可用性)
- 必须在"短期成本(切换)"和"长期成本(中断)"之间做权衡,长期平均自然完成这一权衡

---

### 4.3.2 切换约束

$$\overline{x_{sw}} = \lim_{T\to\infty}\frac{1}{T}\sum_{t=0}^{T-1} \mathbf{1}\{a(t)\ne a(t-1)\} \le C_{\max}$$

**符号定义:**

- $x_{sw}(t) = \mathbf{1}\{a(t)\ne a(t-1)\}$:本时隙是否切换的 0/1 指示
- $\overline{x_{sw}}$:长期平均切换率(每时隙的切换概率)
- $C_{\max}$:切换率上界,默认 $1/120 \,\text{s}^{-1}$ = 平均 2 分钟最多 1 次

**为什么硬约束切换率:**

- 切换在物理上需要天线转向、链路重建(0.3-1 s),频繁切换会让链路始终在"建链"状态
- 切换还触发状态迁移开销(ISL 带宽 + Pre-copy 时间)
- 若不约束,纯优化 $\overline{c}$ 可能让算法每步都换(MPTCP 实测就是这样,切换 70 次/30 min 导致 6.4% 丢包)
- **本约束是 Lyapunov 框架与朴素贪心算法的本质区别**

**$C_{\max} = 1/120$ 怎么来的:**

- LEO 单颗卫星可见窗约 5-15 分钟 → 每 2 分钟最多 1 次切换,保留约 6×~7× 的安全裕度
- 该值在 `config.py:SimConfig.switch_budget_per_sec` 可调

---

### 4.3.3 虚拟队列

$$Q(t+1) = \max\{Q(t) + \mathbf{1}\{a(t)\ne a(t-1)\} - C_{\max}\cdot \Delta t,\;\; 0\}$$

**这是 Lyapunov 框架的核心机制**。每个符号:

| 符号 | 含义 |
|---|---|
| $Q(t)$ | 虚拟队列长度——表示"截至 $t$ 时刻累积的切换预算违反量" |
| $\mathbf{1}\{\cdots\}$ | 本时隙是否真切换了(0/1) |
| $C_{\max} \cdot \Delta t$ | 本时隙允许使用的"切换额度"(预算到账) |
| $\max\{\cdot, 0\}$ | 队列长度不为负 |

**直观解释——用"借贷"类比:**

- 每次真切换 = **借** 1 单位
- 每个时隙过去 = **还** $C_{\max} \cdot \Delta t$ 单位
- $Q(t)$ = 当前**累计欠款**
- $\max\{\cdot, 0\}$ = "还多了不退钱"(余额清零)

**为什么用虚拟队列而不是直接约束:**

- 直接约束(如 $\sum x_{sw} \le N$)是**离线条件**,在线决策时不知道未来,无法判断现在切换是否会触发违约
- 虚拟队列把"全局约束"翻译成"逐时隙的瞬时压力"——当 $Q$ 很大时,算法自然倾向不切换,以"还款"

**理论保证(Neely 2010 经典结果,见 `docs/lyapunov_proof.md` 引理 1):**

若策略让 $Q$ 强稳定($\overline{Q}/t \to 0$),则约束 $\overline{x_{sw}} \le C_{\max}$ 必然满足。

**与代码对应:**

```python
Q = max(Q + (1.0 if is_switch else 0.0) - C_max * dt, 0.0)
```

### 4.3.4 Lyapunov 函数与漂移

#### Lyapunov 函数

$$L(Q(t)) = \frac{1}{2} Q(t)^2$$

**为什么是平方而不是 $|Q|$ 或其它:**

1. **凸 + 光滑**:大 $Q$ 时增长更快,更激进地把队列推回原点,系统更易稳定
2. **数学性质好**:$\Delta(L) = L(Q(t+1)) - L(Q(t))$ 展开后可表示为 $Q(t)$ 的线性项与常数之和,推出闭式策略
3. **控制论标准构造**:Foster-Lyapunov 准则、Lyapunov 稳定性理论的标准选择
4. **系数 $1/2$**:让导数项没有多余的 2 因子,纯粹是公式书写惯例

#### 漂移定义

$$\Delta(Q(t)) \;\triangleq\; \mathbb{E}\{L(Q(t+1)) - L(Q(t)) \mid Q(t)\}$$

**含义:** 给定当前队列长度 $Q(t)$,下一时隙 Lyapunov 函数的期望变化量。

**为什么取条件期望:**

- 决策 $a(t)$ 是确定的,但通用模型下环境(负载 $L$、可见性)可以是随机的
- 条件期望让推导适用于更广场景(包括随机负载)
- 本研究负载是预设序列(无随机性),期望符号退化为恒等,但保留期望便于沿用 Neely 框架

---

### 4.3.5 Drift-Plus-Penalty 上界

$$\Delta(Q(t)) + V\cdot c(t) \;\le\; B + Q(t)\cdot[\mathbf{1}\{a(t)\ne a(t-1)\} - C_{\max}\cdot \Delta t] + V\cdot c(t)$$

**这是 Lyapunov 优化的心脏**。每个符号:

| 符号 | 含义 |
|---|---|
| $\Delta(Q(t))$ | 单时隙漂移(见 4.3.4) |
| $V \cdot c(t)$ | 罚函数项,$V$ 是 utility-延迟权衡参数 |
| $B$ | 与决策**无关**的常数(由 $(x_{sw} - C_{\max}\Delta t)^2 \le 1$ 推出 $B \le 1/2$) |
| 右侧最后两项 | 只与**当下决策** $a(t)$ 有关的部分 |

**怎么推出来的(简版,完整推导在 `docs/lyapunov_proof.md` 第 3 节):**

第一步:由 $\max\{y, 0\}^2 \le y^2$ 得

$$Q(t+1)^2 \le (Q(t) + x_{sw}(t) - C_{\max}\Delta t)^2$$

第二步:展开右侧

$$= Q(t)^2 + (x_{sw} - C_{\max}\Delta t)^2 + 2 Q(t)(x_{sw} - C_{\max}\Delta t)$$

第三步:两边减 $Q(t)^2$,除以 2,取条件期望

$$\Delta(Q(t)) \le \tfrac{1}{2}\mathbb{E}\{(x_{sw} - C_{\max}\Delta t)^2\} + Q(t)\mathbb{E}\{x_{sw} - C_{\max}\Delta t\}$$

第一项就是常数 $B$,第二项是含 $a(t)$ 的项。两边加 $V \cdot c(t)$ 得到 drift-plus-penalty bound。

**关键洞察:** 这个上界**把长期优化问题转化成了逐时隙的瞬时最小化问题**——我们只要每个时隙最小化 RHS,长期目标自然得到约束:

$$\overline{c}^{\text{DPP}} \le \overline{c}^{\text{opt}} + \frac{B}{V}$$

(定理 1,证明见 `docs/lyapunov_proof.md` 第 5 节)

**$V$ 的作用:**

- $V$ **大** → 罚函数 $V \cdot c$ 主导 → 算法**激进**追求低 cost,可能违反 $Q$ 约束(短期 $Q$ 长)
- $V$ **小** → 漂移项主导 → 算法**保守**不切换,$\overline{c}$ 较差但 $Q$ 稳定
- 数学上证明:$\overline{c}$ 误差 $O(1/V)$,队列长度 $O(V)$,这是**经典 utility-延迟权衡**

---

### 4.3.6 闭式决策策略

$$a^*(t) = \arg\min_{a\in\{1,\dots,N\}} \;\; V\cdot \tilde{c}_a(t) + Q(t)\cdot \mathbf{1}\{a \ne a(t-1)\}$$

其中

$$\tilde{c}_a(t) = \alpha\cdot\mathbf{1}\{V_a(t)=0 \vee \Delta T_a(t) < T_h\} + \beta \cdot L_a(t) + \gamma\cdot\mathbf{1}\{a\ne a(t-1)\}$$

**这就是最终落地的策略**。逐项解释:

| 项 | 数学含义 | 操作含义 |
|---|---|---|
| $V \cdot \tilde{c}_a(t)$ | 决策 $a$ 带来的瞬时代价(被 $V$ 放大) | "选这个网关多贵?" |
| $Q(t) \cdot \mathbf{1}\{a \ne a(t-1)\}$ | 当前队列长度 × 切换指示 | "如果切换,队列压力还要再增加,$Q$ 已经很大时再切换是不智的" |

**符号 $\tilde{c}_a$ 的下标 $a$ 是什么意思:**

- $c(t)$ 在 4.1.5 是把实际决策 $a(t)$ 代入的"实际代价"
- $\tilde{c}_a(t)$ 是把**假设决策为 $a$** 代入的"候选代价",用于 argmin 时比较所有候选
- 二者唯一区别:$a$ 是循环变量,$a(t)$ 是最终选定的

**为什么是 $O(N)$ 而非 NP-hard:**

- 决策空间只有 $N$ 个离散候选(本仿真 N=16)
- 每个候选独立计算 score,取 argmin
- 无须求解器(CVXPY 等),纯 NumPy 100 μs 出结果

**与代码对应** (`optimizer/lyapunov_solver.py:lyapunov_online()`):

```python
for a in range(N):
    interrupt = (not vis_k[a]) or (rem[a] < interruption_horizon_sec)
    cost_a = alpha * interrupt + beta * L_k[a]
    switch_indicator = 0.0 if a == current_gw else 1.0
    score = V * (cost_a + gamma_switch * switch_indicator) + Q * switch_indicator
    if score < best_score:
        best_score = score
        best_a = a
```

每一行公式与文档严格一一对应。

### 4.3.7 性能界定理

$$\overline{c}^{\text{Lyap}} \le \overline{c}^{\text{opt}} + \frac{B}{V}$$

$$\overline{Q} \le \frac{B + V\cdot(c_{\max} - c_{\min})}{\epsilon}$$

**含义:** 在 Slater 条件(约束可行有余量 $\epsilon > 0$)下,我们的策略

- **代价**最多比理论最优多 $B/V$,$V$ 越大越接近最优
- **队列长度**(约束违反量)以 $O(V)$ 线性增长

**符号定义:**

- $\overline{c}^{\text{Lyap}}$:本策略的长期平均代价
- $\overline{c}^{\text{opt}}$:**离线最优**长期平均代价(假设未来全可见的"上帝视角")
- $B$:与决策无关的常数,$B \le 1/2$
- $V$:超参,默认 50
- $\epsilon$:Slater 余量(约束的"宽裕度")

**这就是 utility–切换次数权衡的精确数学刻画**——证明完整版见 `docs/lyapunov_proof.md` 第 5 节,共 4 步用"对照策略法":

1. 取 $\pi^{any} = \pi^{opt}$ 推出代价界
2. 取 $\pi^{any} = \pi^{slack}$(Slater 条件下的松弛策略)推出队列界
3. Foster-Lyapunov 准则把瞬时漂移上界翻译成长期队列上界
4. 伸缩求和得到时间平均结果

**实测补充:** 本仿真场景下 $V \in [1, 1000]$ 给出相同决策——决策被可见性硬约束主导。这一**对 V 的不敏感性**是 Lyapunov 在卫星场景下工程鲁棒性强的有力证据,详见 `docs/lyapunov_proof.md` 第 8 节。

## 设计哲学小结

| 维度 | 本方案选择 | 替代方案及缺陷 |
|---|---|---|
| **时间建模** | 离散时隙 + 长期平均 | 连续控制 → 不必要复杂;瞬时最小 → 短视贪心 |
| **可见性** | 二值 $V_i$ + 剩余时长 $\Delta T_i$ | 只用瞬时信号强度 → 失去星历可预测性的优势 |
| **代价函数** | 加权和(中断+负载+切换)| 单一目标 → 偏废一面 |
| **约束** | 长期平均切换率 + 虚拟队列 | 硬约束 → 在线不可行;无约束 → 过度切换(MPTCP 反模式)|
| **决策算法** | drift-plus-penalty 闭式 argmin | CVXPY 求解 → 慢;DRL → 不稳定无保证 |
| **状态分类** | $C_{static}$ vs $C_{dynamic}$ | 不分 → 要么全量频繁迁移(慢)、要么完全不迁移(丢) |
| **分片缓存** | 增量 `_complete_set` 维护 | 全表扫描 → O(N²),200× 慢 |
| **决策权重** | $\alpha=1 > \gamma=0.5 > \beta\cdot \max L=0.3$ | 顺序不当 → 负载小波动触发切换,反模式 |

每一个看似"普通"的建模选择,背后都对应:

1. **卫星场景特定难点**(星历可知、ISL 时变、协议级状态)
2. **数学可解性**(闭式 argmin、O(N) 复杂度、性能界证明)
3. **工程可实现性**(0.5 ms 决策、SimPy 仿真可跑通)

三方权衡的综合产物。
